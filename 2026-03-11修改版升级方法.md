# 2026-03-11修改版升级方法

## 一、升级前准备

1. 停止当前系统（后端服务、MariaDB 服务、前端访问）。
2. 备份以下目录（必须）：
   - `backend/config.json`
   - `backend/data/`（如使用本地数据目录）
   - `backend/uploads/`
   - `backend/archives/`
3. 记录当前访问地址与端口（默认 `http://<IP>:3002`）。

## 二、获取升级包

本次重新构建后的发布目录为：

- `deploy_bundle/`

其中包含：

- `deploy_bundle/backend/`
- `deploy_bundle/frontend/`
- `deploy_bundle/README.txt`
- `deploy_bundle/quick-start.bat`

## 三、执行升级（覆盖式）

1. 将新版本 `deploy_bundle` 拷贝到目标机器临时目录。
2. 进入现网安装目录，先确认服务已停止。
3. 按以下规则覆盖文件：
   - 覆盖 `backend/`（保留你现网 `config.json` 和业务数据目录）
   - 覆盖 `frontend/`
4. 若你使用便携部署方式，可执行：
   - `backend/check-and-setup-env.bat`
5. 启动后端服务，确认服务监听正常。

## 四、与本次修改相关的重点验证

1. 居民查询页：
   - 状态默认值为“正常”；
   - 点击“重置”后状态仍保持“正常”；
   - 新增筛选“政治面貌”“家庭地址”可生效；
   - 查询框布局为更紧凑两行。
2. 人口死亡上报：
   - 查询结果不应混入 `active` 状态人员。
3. 字典管理：
   - “使用单位”可见；
   - 打印预览可显示填报单位与村名。
4. 机构管理：
   - 民族字段、政治面貌字段正常显示。
5. IP 访问：
   - 图标与图片可正常显示。

## 五、回滚方案

若升级异常，按以下顺序回滚：

1. 停止新版本服务；
2. 用升级前备份还原 `backend/` 与 `frontend/`；
3. 还原 `backend/config.json` 与数据目录；
4. 重启服务并复测关键页面。

## 六、备注

本次升级包已基于 `2026-03-11` 代码重新构建前端产物并重建 `deploy_bundle`。  
建议升级后先做一轮关键业务冒烟测试，再开放给全员使用。
