村委会管理系统（Windows + Node.js + MariaDB + Nginx）部署说明

适用对象：普通用户/非开发人员
目标：在一台 Windows 电脑上从零部署并能正常访问系统
默认安装盘：D:\（可按实际情况改）

----------------------------------------
一、你需要提前准备的东西（必须做）

1) 操作系统
- Windows 10/11（64 位）
- 需要有管理员权限（安装软件、配置端口）

2) 部署包
- 已包含前端产出物 + 后端源码 + node_modules
- 例如：D:\deploy_bundle

3) 需要下载安装的软件（必须）
- Node.js（LTS 版本）
- MariaDB（10.6+）
- Nginx（Windows 版，解压即可）

下载来源（建议从官网或官方发布页下载）
- Node.js：nodejs.org
- MariaDB：mariadb.org
- Nginx：nginx.org
- 可选 NginxUI：在 GitHub Releases 搜索 “NginxUI”

----------------------------------------
二、一步步部署（按顺序执行）

第 1 步：安装 Node.js
- 安装路径建议：D:\nodejs
- 安装后必须把 D:\nodejs 加到系统 PATH
- 验证：打开命令行输入 node -v，能看到版本号

第 2 步：安装 MariaDB
- 建议安装路径：D:\MariaDB
- 默认端口：3306
- 设置 root 密码并牢记
- 安装后验证：打开命令行输入 mysql -u root -p

第 3 步：解压 Nginx
- 解压到 D:\nginx（或你自定义目录）
- Nginx 不需要安装，只要解压即可

第 4 步：把部署包放到目标机器
- 例如：D:\deploy_bundle
- 确保目录结构如下：
  D:\deploy_bundle\frontend\
  D:\deploy_bundle\backend\

第 5 步：初始化数据库（推荐自动脚本）
进入目录：D:\deploy_bundle\backend
双击运行：setup-local-mariadb.bat
- 按提示输入 root 密码、数据库名、业务用户名
- 脚本会：创建数据库、创建业务账号、可选导入初始化数据
- 脚本完成后会输出数据库密码（一定要记录下来）

第 6 步：设置数据库密码环境变量（必须）
部署环境不再允许 config.json 明文密码，必须用环境变量 DB_PASSWORD。
你可以用下面任意一种方式：

方式 A：临时设置（只对当前命令窗口有效）
- CMD:
  set DB_PASSWORD=你记录的密码
- PowerShell:
  $env:DB_PASSWORD = "你记录的密码"

方式 B：永久设置（重启电脑后仍有效）
- CMD:
  setx DB_PASSWORD "你记录的密码"

说明：如果你不想手动设置，直接运行后端启动脚本时会提示输入（不回显）。

第 7 步：启动后端服务
进入目录：D:\deploy_bundle\backend
推荐方式（会提示输入 DB_PASSWORD）：
- start.bat   或   start-backend.bat

如果你已手动设置环境变量，可直接：
- npm run start

启动成功后应看到：
- 后端服务运行在 http://localhost:3002

第 8 步：配置 Nginx 反向代理（前端访问用）
目的：让浏览器访问 http://localhost/ 就能打开前端，并把 /api 请求转发到后端。

最简单方式：修改 Nginx 配置（示例）
- 打开 D:\nginx\conf\nginx.conf
- 找到 server 段，替换为：

server {
  listen 80;
  server_name localhost;
  root D:/deploy_bundle/frontend;
  index index.html;

  client_max_body_size 50m;

  location / {
    try_files $uri $uri/ /index.html;
  }

  location /api/ {
    proxy_pass http://127.0.0.1:3002;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }

  location /uploads/ {
    proxy_pass http://127.0.0.1:3002;
  }

  location /archives/ {
    proxy_pass http://127.0.0.1:3002;
  }
}

重载 Nginx：
- D:\nginx\nginx.exe -s reload

第 9 步：访问验证
- 打开浏览器访问：http://localhost/
- 若能打开登录页并登录成功，部署完成

----------------------------------------
三、常见问题排查

1) 浏览器打不开（80 端口冲突）
- 现象：启动 Nginx 后访问失败
- 解决：把 listen 80 改成 8080，然后访问 http://localhost:8080

2) 登录失败或接口报错
- 检查后端是否启动
- 确认 DB_PASSWORD 是否设置正确
- 确认数据库是否已初始化成功

3) 报错找不到模块
- 进入 backend 目录执行：npm rebuild

4) 数据库连接失败
- 确认 MariaDB 服务是否已启动
- 确认 DB_HOST/DB_USER/DB_NAME/DB_PORT 是否正确（可选设置）

----------------------------------------
四、你还需要做的额外工作（总结）

- 安装 Node.js / MariaDB / Nginx
- 配置系统 PATH（至少 Node.js）
- 运行数据库初始化脚本
- 设置环境变量 DB_PASSWORD
- 启动后端服务
- 配置并重载 Nginx

完成以上步骤后系统即可对外提供访问。
