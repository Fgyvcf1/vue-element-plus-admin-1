部署步骤（简版）

备份数据库 + 备份旧 backend/uploads
停止后端服务（或杀掉占用 3002 的进程）
用新包的 backend/ 覆盖旧后端目录
保留旧的：backend/uploads、backend/config.json、数据库数据目录
执行数据库升级脚本
mysql -u root -p village < db/migrations/2026-03-06-mediation-updates.sql
启动后端
node app.js
如果 3002 端口占用：node kill-port.js 3002 后再启动
说明

前端已构建并放进 backend/frontend/，目标机不需要再 pnpm install 或 pnpm build。
我未包含 backend/uploads，避免覆盖你服务器的附件数据。
